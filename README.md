# AlmaPay

AlmaPay is a deployment and operations toolkit for self-hosted [BTCPay Server](https://btcpayserver.org/) on Alma Linux and other RHEL-family distributions. The goal is reproducible, automated setup using rootless Podman and Podman Compose, with TLS terminated on the host by a reverse proxy (Caddy is the reference choice). The stack is intended to support on-chain Bitcoin, on-chain Monero, Lightning via the Boltz plugin in nodeless mode, and card payments via the Stripe plugin, with integrators talking to BTCPay through the Greenfield API and HTTPS webhooks rather than direct access to chain daemons or payment providers.

The deployment model is a single Podman Compose project made of dedicated containers, not one monolithic container. BTCPay Server, PostgreSQL, and NBXplorer stay together as the application tier; optional local `bitcoind` (pruned) and Monero services (`monerod`, wallet RPC) run as separate containers in the same compose graph when you want self-hosted chain indexing. Boltz and Stripe are BTCPay plugins and do not require their own hosts. Caddy (or an equivalent reverse proxy) runs outside the compose stack on the host, binding public HTTPS and forwarding to BTCPay on a loopback high port (for example `127.0.0.1:8080`). Containers run under a dedicated unprivileged system user with linger and a user systemd unit so the stack survives reboot without rootful Podman or privileged capabilities.

Chain nodes may remain in the same compose project for simplicity, or be split onto separate machines or replaced with external RPC providers when disk, sync time, or blast radius matter more than single-host operation. Plan generously for a full local layout: on the order of 4 vCPU, 8 GB RAM, and 400 GB SSD, with pruned Bitcoin using roughly 50 GB and a Monero full node commonly exceeding 150 GB and growing over time. Monero wallet configuration is server-wide in BTCPay; stores that must not share Monero treasury or view-key access need a separate BTCPay deployment, not just another store on the same instance.

This repository is early work in progress. It will codify host bootstrap, compose generation (via upstream [btcpayserver-docker](https://github.com/btcpayserver/btcpayserver-docker) fragments), environment templates, systemd persistence, backup guidance, and update procedures for the layout above.
